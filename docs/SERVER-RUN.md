# تشغيل السيرفر

## التشغيل العادي

```bash
npm install
node server.js
```

أو:

```bash
npm start
```

يجب أن تظهر: `🚀 Server running on http://localhost:3000`

---

## إذا لم يعمل السيرفر

### 1. المنفذ مستخدم (Port already in use)

**الرسالة:** `Port 3000 is already in use`

**الحل:**
- أغلق أي نافذة طرفية أخرى تشغّل السيرفر (Ctrl+C)، أو
- غيّر المنفذ: أنشئ أو عدّل ملف `.env` وأضف:
  ```
  PORT=3001
  ```
  ثم شغّل من جديد: `node server.js`

### 2. وحدة غير موجودة (Cannot find module)

**الرسالة:** `Cannot find module 'better-sqlite3'` أو `bcrypt` أو غيرها

**الحل:**
```bash
npm install
```
ثم `node server.js` مرة أخرى.

### 3. خطأ في قاعدة البيانات (Database error)

**الرسالة:** `Database error: ...`

- تأكد أن مجلد `client/data` موجوداً.
- إذا ظهر خطأ يتعلق بـ `better-sqlite3` على Windows، قد تحتاج إلى **Build Tools** لـ Node (مثلاً من [windows-build-tools](https://github.com/felixrieseberg/windows-build-tools) أو Visual Studio Build Tools).

### 4. الموقع يعمل في متصفح البرنامج (Cursor) ولا يعمل في Chrome / Edge / Firefox

**الحلول:**

1. **استخدم نفس الرابط في المتصفح العادي**  
   بعد تشغيل السيرفر، افتح في المتصفح:
   - `http://localhost:3001` (إذا كان المنفذ 3001 في `.env`)
   - أو `http://127.0.0.1:3001`  
   تأكد أن الرابط يبدأ بـ `http://` وليس `https://`.

2. **امسح الكاش (Cache)**  
   في المتصفح: Ctrl+Shift+Delete → اختر "الصور والملفات المخزنة مؤقتاً" → مسح. ثم أعد تحميل الصفحة (F5 أو Ctrl+F5).

3. **تحقق من وحدة التحكم (Console)**  
   في المتصفح اضغط F12 → تبويب **Console**. إذا ظهرت أخطاء حمراء (مثلاً Failed to load، أو CORS، أو 404)، انسخها لمعرفة السبب.

4. **جرب وضع التصفح الخاص أو عطّل الإضافات**  
   بعض الإضافات (حظر الإعلانات، الخصوصية) قد تمنع طلبات `localhost`. جرّب نافذة خاصة (Incognito/Private) أو عطّل الإضافات لموقع `localhost`.

5. **السيرفر يستمع على جميع الواجهات**  
   في المشروع تم ضبط السيرفر ليعمل على `0.0.0.0` حتى يعمل من أي متصفح على الجهاز. إذا غيّرت `HOST` في `.env` تأكد أنك تفتح الرابط الصحيح.

### 5. لماذا يجب مسح Cookies وبيانات الموقع لرؤية التعديلات؟

الموقع يستخدم **Service Worker** (PWA) لتخزين الصفحات والملفات مؤقتاً. المتصفح كان يعيد استخدام النسخة المخزنة بدل تحميل الملفات المحدّثة.

**ما تم عمله:** على **localhost** لا يتم تسجيل الـ Service Worker، لذلك أثناء التطوير المحلي تظهر التعديلات فوراً دون الحاجة لمسح أي شيء. الـ Service Worker يعمل فقط عندما يفتح المستخدمون الموقع على عنوان غير localhost (مثلاً بعد النشر).

**إذا كنت تختبر على عنوان غير localhost** وتريد رؤية التعديلات دون مسح البيانات: في المتصفح افتح F12 → تبويب Application → Service Workers → فعّل "Update on reload" أو "Bypass for network".

### 6. أي خطأ آخر

انسخ رسالة الخطأ كاملة من الطرفية (Terminal) وشاركها لتحليل المشكلة.
