# تفعيل أو تعطيل كرت الشاشة (GPU) في Cursor

إعدادات الـ GPU **لا تظهر** في قائمة الإعدادات (Settings) العادية. توجد في **معطيات التشغيل (Runtime Arguments)**.

---

## الخطوات

### 1. فتح معطيات التشغيل
1. اضغط **Ctrl+Shift+P** (أو من القائمة: File → Preferences → Command Palette).
2. اكتب: **Configure Runtime Arguments**
3. اختر: **Preferences: Configure Runtime Arguments**

سيُفتح ملف اسمه **argv.json** (قد يكون فارغاً أو فيه سطور معطّلة بتعليق).

### 2. التأكد من تفعيل كرت الشاشة (الوضع الافتراضي)
- **لتشغيل Cursor بكفاءة باستخدام الـ GPU (الوضع الموصى به):**  
  تأكد أن السطر التالي **غير موجود** أو **معطّل** (مسبوق بـ `//`):
  ```json
  // "disable-hardware-acceleration": true
  ```
  إذا كان السطر موجوداً **بدون** `//`، غيّره إلى:
  ```json
  "disable-hardware-acceleration": false
  ```
  أو احذف السطر تماماً.

- **لتعطيل كرت الشاشة** (إذا واجهت تجمّد أو استهلاك عالي للـ GPU):
  ```json
  "disable-hardware-acceleration": true
  ```

### 3. إعادة تشغيل Cursor
بعد أي تعديل على **argv.json** أغلق Cursor بالكامل ثم افتحه من جديد.

---

## ملاحظات
- Cursor (مثل VS Code) يستخدم الـ GPU افتراضياً؛ لا تحتاج لتفعيل شيء إضافي في أغلب الأجهزة.
- إذا كان البرنامج يعمل ببطء أو يتجمّد، جرّب **تعطيل** التسريع بدل تفعيله (بعض كروت الشاشة أو drivers تسبب مشاكل).
- لتفضيل استخدام كرت الشاشة المخصص على Windows: إعدادات Windows → النظام → شاشة → إعدادات الرسومات → اختر Cursor → خيارات → **أداء عالي**.

## خطط مستقبلية (للعمل عليها)

- توثيق أي إعدادات إضافية لـ Cursor تتعلق بالأداء أو الذاكرة إن ظهرت في الإصدارات اللاحقة.
