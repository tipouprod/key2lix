# تذكير بالسلة المهجورة (N5)

## نظرة عامة

- **جدول:** `abandoned_cart` (client_id، email، items_json، updated_at، reminded).
- **API:** `POST /api/cart` — يحفظ السلة للعميل المسجّل (من الجلسة) أو للزائر عند إرسال `email` مع `items`.
- **سكربت:** `scripts/abandoned-cart-reminder.js` — يختار سلات لم تُحدّث منذ 24 ساعة (قابل للتعديل)، يرسل بريد تذكير ثم يعلّم السلة كمُذكَّر بها.

## متغيرات البيئة

| المتغير | الوصف |
|--------|--------|
| `SMTP_*` | مطلوب لإرسال البريد (نفس إعدادات الإشعارات). |
| `SITE_URL` أو `BASE_URL` | الرابط الأساسي للموقع (مثل `https://key2lix.com`) لرابط السلة في البريد. |
| `ABANDONED_CART_HOURS` | عدد الساعات بعد آخر تحديث للسلة لإرسال التذكير (افتراضي 24). |
| `ABANDONED_CART_LIMIT` | أقصى عدد سلات تُعالج في كل تشغيل (افتراضي 50). |

## الجدولة (cron)

تشغيل السكربت مرة يومياً، مثلاً الساعة 10 صباحاً:

```bash
0 10 * * * cd /path/to/key2lix && node scripts/abandoned-cart-reminder.js
```

أو عبر npm:

```bash
0 10 * * * cd /path/to/key2lix && npm run abandoned-cart-reminder
```

## الواجهة الأمامية

مُفعّلة في المشروع:

1. **عميل مسجّل:** تُرسل السلة تلقائياً عند فتح صفحة السلة، وعند الإضافة من صفحة المنتج، وعند الحذف أو التفريغ من صفحة السلة (`Key2lixCart.syncToServer()`).
2. **زائر:** في صفحة السلة يظهر صندوق «احصل على تذكير إذا تركت السلة» مع حقل بريد وزر «حفظ للتذكير»؛ عند الإدخال يُستدعى `POST /api/cart` مع `email` و `items`.

مرجع الطلب في [API.md](API.md#السلة-المهجورة-n5).
