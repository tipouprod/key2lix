# القفزة للمستوى الأكبر — Key2lix

خارطة طريق لرفع الموقع لمستوى احترافي: أداء، موثوقية، وقابلية توسع.

---

## ملاحظة

المراحل السابقة (الأساس، API وبيانات، الواجهة والتجربة، الإنتاج والمراقبة) تم تنفيذها — ضغط، rate limit، SQLite، كاش، تحقق المدخلات، نسخ احتياطي، PWA، إمكانية وصول، Docker، CI/CD، سجلات، Sentry، خطوة البناء.

---

## خطط مستقبلية (للعمل عليها)

| # | المجال | الخطوة | ملاحظات |
|---|--------|--------|----------|
| 1 | **الأداء** | Brotli بدل Gzip حيث يدعمه العميل | تقليل أكبر لحجم النقل |
| 2 | **الكاش** | كاش Redis أو مشابه لجلسات أو بيانات متكررة | عند نمو التحميل |
| 3 | **البيانات** | ترحيل اختياري إلى PostgreSQL عند الحاجة لتوسع أفقي | SQLite كافٍ حالياً |
| 4 | **المراقبة** | لوحة بسيطة لصحة السيرفر (health check) ووقت الاستجابة | راجع [MONITORING.md](MONITORING.md) |
| 5 | **الاختبارات** | اختبارات وحدة لـ API واختبارات E2E لمسارات الطلب والدخول | Jest + supertest أو Playwright |
| 6 | **الأمان** | تفعيل CSP (Helmet) تدريجياً وتأمين الكوكيز في الإنتاج | `secure: true`, `sameSite` |

يمكن تنفيذ هذه النقاط حسب الأولوية والحاجة.
