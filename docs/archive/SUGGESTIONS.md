# اقتراحات تطوير Key2lix

قائمة مقترحات منظمة حسب الأولوية والفئة. اختر ما يناسبك ونفّذه خطوة بخطوة.

> **اقتراحات احترافية وقوية (أحدث):** راجع [SUGGESTIONS-PRO.md](SUGGESTIONS-PRO.md) لخطة مقترحات محدّثة مع الأثر والجهد وأولويات تنفيذ وحالة التنفيذ.

---

## واجهة وتجربة المستخدم (UX/UI)

| # | الاقتراح | الوصف |
|---|----------|--------|
| 1 | **توحيد تصميم الصفحات الثانوية** | صفحات مثل Contact، How to buy، How to sell، Support، Partnership، Key2lix Plus، News — إعطاؤها نفس أسلوب الـ index (عنوان واضح، خلفية، أزرار بنفسجية، نفس الـ footer/navbar). |
| 2 | **تحسين صفحة تسجيل الدخول** | تصميم صندوق تسجيل أنظف (لوجو، حقول، زر، رسالة خطأ واضحة)، مع دعم الثيم الفاتح/الداكن. |
| 3 | **تحسين الفوتر** | مراجعة الروابط (سياسة الخصوصية، الشروط، وسائل التواصل إن وُجدت)، وإضافة أيقونات أو أقسام صغيرة. |
| 4 | **رسائل تحميل (Loading)** | عرض "جاري التحميل..." أو skeleton عند جلب المنتجات في صفحات المنتجات/الفئة بدل شاشة فارغة. |
| 5 | **زر "العودة للأعلى" في كل الصفحات** | حالياً قد يكون في الـ index فقط — إضافته عبر common.js أو الفوتر ليعمل في كل الصفحات. |

## تقني و SEO

| # | الاقتراح | الوصف |
|---|----------|--------|

## خطط مستقبلية (للعمل عليها)

اعتماداً على الجداول أعلاه، يمكن اختيار ما يلي كخطة عمل (ما لم يكن مذكوراً في «ما تم تنفيذه» أدناه):

- **واجهة وتجربة:** (1) توحيد تصميم الصفحات الثانوية، (2) تحسين صفحة الدخول، (3) تحسين الفوتر، (5) زر العودة للأعلى في كل الصفحات.
- **ميزات:** (6) فلترة وبحث في صفحة المنتجات، (7) ترتيب المنتجات، (8) إشعار بعد الإرسال، (9) صفحة عروض اليوم/تخفيضات.
- **تقني و SEO:** (11) تحديث Sitemap و robots، (12) Meta ديناميكي للمنتج (إن لم يُنفَّذ بالكامل)، (13) تحسين سرعة التحميل، (14) صفحة 404 مخصصة.
- **محتوى وترجمة:** (15) ترجمة FAQ، (16) نصوص الصفحات الثانوية، (17) مفاتيح ترجمة ناقصة.
- **أدمن:** (18) تحسين جداول الأدمن، (20) تقييد محاولات تسجيل الدخول.
- **نشر وتسويق:** (21) ربط النشر بـ Netlify، (22) نشرة بريد، (23) Google Analytics أو بديل.

## اقتراحات مطورة (للمطورين)

اقتراحات تقنية أعمق لتحسين البنية، الأداء، والأمان — تنفّذ بعد استقرار الميزات الأساسية.

### أداء وبنية تحتية

| # | الاقتراح | الوصف |
|---|----------|--------|
| A1 | **ضغط الاستجابة (Gzip/Brotli)** | تفعيل `compression` في Express لضغط HTML/CSS/JS والحد من حجم النقل. |
| A2 | **كاش للملفات الثابتة** | إرسال رؤوس `Cache-Control` و `ETag` للملفات في `/assets` لتحسين التحميل المتكرر. |
| A3 | **تحسين الصور تلقائياً** | سكربت أو pipeline لضغط/تحويل الصور إلى WebP مع fallback، أو استخدام خدمة (مثل Cloudinary) عند الرفع. |
| A4 | **تقسيم JS (Code splitting)** | فصل سكربتات الصفحات (مثلاً تحميل `form.js` فقط في صفحة الطلب) لتقليل الحجم الأولي. |

### API والسيرفر

| # | الاقتراح | الوصف |
|---|----------|--------|
| A5 | **نسخة للـ API (API versioning)** | مسارات مثل `/api/v1/order` لتمكين تغييرات مستقبلية دون كسر العملاء. |
| A6 | **حد معدل الطلبات (Rate limiting)** | تطبيق rate limit عام (مثلاً `express-rate-limit`) بخلاف حد تسجيل الدخول، لحماية من الإغراق. |
| A7 | **التحقق من صحة المدخلات** | استخدام مكتبة (مثل `joi` أو `express-validator`) للتحقق من جسم طلبات الـ API قبل المعالجة. |
| A8 | **سجلات منظمة (Structured logging)** | استخدام logger (مثل `pino` أو `winston`) مع مستويات (info, error) وتنسيق JSON لتسهيل المراقبة. |

### البيانات والأمان

| # | الاقتراح | الوصف |
|---|----------|--------|
| A9 | **قاعدة بيانات بدل JSON** | عند نمو البيانات: نقل الطلبات والرسائل والمنتجات إلى SQLite أو PostgreSQL مع حفظ الـ JSON كنسخة احتياطية أو تصدير. |
| A10 | **نسخ احتياطي تلقائي** | كرون أو سكربت ينسخ `client/data/*.json` (والـ backup الحالي) إلى مجلد أو سحابة بشكل دوري. |
| A11 | **تفعيل CSP (Content Security Policy)** | إعداد رؤوس Helmet CSP تدريجياً (السماح للنطاق، CDN، inline مقتصد) لتحسين الأمان. |
| A12 | **تأمين جلسة الإدارة** | استخدام `cookie: { secure: true }` في الإنتاج، وربما `sameSite: 'strict'` وتجديد session بعد فترة. |

### الواجهة الأمامية

| # | الاقتراح | الوصف |
|---|----------|--------|
| A13 | **PWA (Progressive Web App)** | إضافة `manifest.json` و Service Worker بسيط للعمل دون اتصال (مثلاً الصفحة الرئيسية + المنتجات المخزنة مؤقتاً). |
| A14 | **تحسين إمكانية الوصول (a11y)** | مراجعة ARIA، تباين الألوان، التنقل بلوحة المفاتيح، واختبار بقارئ الشاشة. |
| A15 | **Core Web Vitals** | قياس LCP, FID, CLS (مثلاً عبر Lighthouse أو Real User Monitoring) وتحسين الصور والتخطيط وفقها. |
| A16 | **بيئة بناء (Build step)** | استخدام أداة (Vite، Parcel، أو webpack بسيط) لدمج/تصغير CSS و JS وربما دعم ES modules. |

### الاختبار والنشر

| # | الاقتراح | الوصف |
|---|----------|--------|
| A17 | **اختبارات آلية** | اختبارات وحدة لـ API (مثلاً Jest + supertest) واختبار E2E للطلبات الرئيسية (Playwright أو Cypress). |
| A18 | **Docker** | `Dockerfile` لتشغيل السيرفر في حاوية، مع إمكانية docker-compose لاحقاً (سيرفر + قاعدة بيانات). |
| A19 | **CI/CD** | تشغيل الاختبارات والـ lint على كل commit (GitHub Actions أو غيره) وربط النشر بـ Netlify/Render تلقائياً. |
| A20 | **تتبع الأخطاء** | دمج خدمة (مثل Sentry) لالتقاط أخطاء الجافاسكربت وأخطاء السيرفر في الإنتاج. |

### خطط مستقبلية (مرحلة مطورة)

- **أداء:** A3 تحسين الصور تلقائياً (WebP)، A4 تقسيم JS (Code splitting). — **تم:** A3 عبر sharp عند الرفع؛ A4 تحميل search.js مؤجّل.
- **أمان:** A11 تفعيل CSP، A12 تأمين جلسة الإدارة. — **تم:** CSP في Helmet؛ cookie secure/sameSite.
- **اختبار ونشر:** A17 اختبارات آلية، تحسين CI/CD حسب الحاجة. — **تم:** Jest + supertest في `tests/`، CI يشغّل `npm test`.
- **مراقبة:** A20 تتبع أخطاء (Sentry) — **تم:** سيرفر + عميل (راجع MONITORING.md).

## ما تم تنفيذه في المشروع (مرجع سريع)

| الفئة | المنفّذ |
|-------|--------|
| أداء | A1 ضغط، A2 كاش، (4) رسائل تحميل، (13) lazy للصور؛ **A3** تحويل الصور المرفوعة إلى WebP (sharp)؛ **A4** تحميل search.js مؤجّل (requestIdleCallback) لصفحات ذات data-key2lix-search |
| أمان | **A11** تفعيل CSP في Helmet (script/style/img/connect-src…)؛ **A12** تأمين جلسة الإدارة (cookie: secure في الإنتاج، sameSite: lax) |
| API | A5 نسخة API، A6 rate limit، A7 تحقق المدخلات |
| بيانات | A9 SQLite للطلبات والرسائل والمنتجات، A10 نسخ احتياطي + جدولة |
| واجهة | A13 PWA (manifest + Service Worker)، (5) زر العودة للأعلى، (10) نسخ رابط المنتج، (12) meta ديناميكي للمنتج |
| نشر | A18 Docker، A19 CI (GitHub Actions)، (11) Sitemap/robots، (14) 404؛ **A17** اختبارات آلية (Jest + supertest لـ API) مضافة في `tests/` و CI يشغّل `npm test` |
| أدمن | تبويب الموردون، تبويب الأرباح، إجمالي العمولات، (19) تأكيد قبل الحذف، (20) حد محاولات الدخول |
| مراقبة | سجلات pino؛ **A20** Sentry سيرفر + عميل (GET /api/config، تحميل Sentry للمتصفح من common.js) — راجع [MONITORING.md](MONITORING.md) |
| إمكانية وصول | ARIA، :focus-visible، رابط تخطي، معالم main/footer/nav، صفحة هيرو موحدة للثانوية |
| عميل وسلة | سلة مشتريات (localStorage)، صفحة /cart، أضف للسلة من صفحة المنتج؛ تسجيل دخول عميل (تسجيل، دخول، حسابي، طلباتي)، ربط الطلب بـ client_id، تعبئة تلقائية في نموذج الطلب |
| الصفحة الرئيسية | شريط بحث، بانر شحن مجاني، شريط طرق الدفع، إحصائيات متحركة، آخر طلب (Social Proof)، Sticky CTA، تابات عروض، معاينة سريعة، ربط المنصات بالفئات — راجع [SUGGESTIONS-PRO.md](SUGGESTIONS-PRO.md) |
