# اقتراحات مستقبلية — صفحة العميل (حسابي)

وثيقة مقترحات لتطوير صفحة العميل `/client-account` ومسارات التسجيل والدخول ومرتبطة بحساب العميل.

---

## الوضع الحالي (ملخص)

| الميزة | الحالة |
|--------|--------|
| تسجيل الدخول / إنشاء حساب | صفحة تسجيل، دخول، تحقق من البريد برقم سري (OTP) |
| حسابي (ضيف) | رسالة "سجّل الدخول" مع روابط تسجيل الدخول والشراء والرئيسية |
| ترحيب وإحصائيات | اسم المستخدم + عدد الطلبات (كل / قيد التنفيذ / مكتمل) |
| تأكيد البريد | تنبيه + حقل إدخال الرقم السري المرسل بالبريد، صلاحية 15 دقيقة |
| روابط سريعة | جميع المنتجات، الدعم، المفضلة |
| طلباتي | جدول مع فلتر (الكل / قيد التنفيذ / مكتمل)، محادثة، تحميل PDF الفاتورة |
| المفضلة | صفحة منفصلة `/wishlist` مرتبطة بالحساب |
| تقديم الطلب | يتطلب تسجيل دخول + بريد مؤكد؛ البيانات من الحساب (للقراءة فقط) |

## 1. الملف الشخصي والإعدادات

| الاقتراح | الوصف | أولوية | الحالة |
|----------|--------|--------|--------|
| **تبويب "ملفي" أو "إعداداتي"** | تعديل الاسم، الهاتف، البريد (مع إعادة التحقق إن لزم)، العنوان من داخل حسابي. | متوسطة | ✅ منفّذ: قسم «ملفي» مع الاسم، الهاتف، العنوان (PATCH /api/client/me) |
| **تغيير كلمة المرور** | زر أو قسم "تغيير كلمة المرور" داخل حسابي بدل الاعتماد على صفحة منفصلة فقط. | قصيرة | ✅ منفّذ: قسم تغيير كلمة المرور في حسابي (POST /api/client/change-password) |
| **عنوان افتراضي للتوصيل** | حفظ عنوان واحد أو أكثر (منزل، عمل) واختياره عند الطلب. | متوسطة | ✅ منفّذ: حقل عنوان في ملفي يُحفظ ويُستخدم عند الطلب |
| **عناوين متعددة محفوظة** | إدارة قائمة عناوين مع اسم (مثل "المنزل"، "العمل") واستخدامها في الطلبات. | طويلة | |
| **تفضيلات اللغة والوضع** | حفظ لغة الواجهة (عربي/إنجليزي) ووضع التصفح (فاتح/داكن) مرتبطة بالحساب إن رغبت. | منخفضة | |

## 2. الطلبات وتجربة الشراء

| الاقتراح | الوصف | أولوية |
|----------|--------|--------|
| **زر "إعادة الطلب"** | من جدول الطلبات: زر يعيد إضافة نفس المنتج (بنفس الفئة/السعر) إلى السلة أو يفتح صفحة الطلب. | قصيرة |
| **بحث في طلباتي** | حقل بحث (برقم الطلب، اسم المنتج، تاريخ) مع تصفية فورية. | متوسطة |
| **ترقيم الصفحات أو "تحميل المزيد"** | عند كثرة الطلبات: تقسيم الصفحة أو تحميل تدريجي. | متوسطة |
| **تصدير طلباتي** | تصدير قائمة الطلبات (CSV/Excel) أو طباعة ملخص. | متوسطة |
| **تقييم الطلب بعد الإكمال** | بعد اكتمال الطلب: دعوة لتقييم المنتج/البائع من حسابي أو من صفحة الطلب. | متوسطة |
| **تتبع التوصيل** | إن وُجد رقم تتبع من البائع: عرضه في تفاصيل الطلب مع رابط شركة الشحن. | طويلة |
| **فواتير مجمعة** | تحميل فاتورة شهرية أو لفترة محددة تجمع عدة طلبات. | منخفضة |

## 3. الإشعارات والتواصل

| الاقتراح | الوصف | أولوية | الحالة |
|----------|--------|--------|--------|
| **إشعارات داخل الموقع** | جرس إشعارات في الـ navbar للعميل (رد على طلب، تحديث حالة طلب، عرض جديد). | متوسطة | ✅ منفّذ: جرس في الـ navbar عند تسجيل دخول العميل، إشعارات (رد جديد، إكمال طلب) مع احترام تفضيل «داخل الموقع» |
| **تفضيلات الإشعارات** | اختيار تلقي التنبيهات: داخل الموقع فقط، أو أيضاً بالبريد (تحديثات الطلبات، عروض). | متوسطة | ✅ منفّذ: قسم في حسابي (إشعارات داخل الموقع + البريد)، PATCH /api/client/me، البريد يُرسل فقط إن تفعيل العميل «إشعارات بالبريد» |
| **تنبيه "رد جديد على طلبك"** | إشعار أو بريد عند رد البائع/الأدمن في محادثة الطلب. | منخفضة (قد يكون منفذاً جزئياً) | ✅ منفّذ: إشعار داخل الموقع + بريد (حسب تفضيلات العميل) عند رد البائع في محادثة الطلب |

## 5. الواجهة وتجربة المستخدم (UX)

| الاقتراح | الوصف | أولوية | الحالة |
|----------|--------|--------|--------|
| **تبويبات في حسابي** | تنظيم المحتوى: طلباتي | المفضلة | ملفي | إعدادات (بدل عرض كل شيء في صفحة واحدة). | متوسطة | ✅ منفّذ: أربعة تبويبات (طلباتي، المفضلة، ملفي، إعدادات) مع تبديل المحتوى |
| **حفظ آخر فلتر طلبات** | تذكر آخر فلتر مستخدم (الكل/قيد التنفيذ/مكتمل) في localStorage. | قصيرة | ✅ منفّذ: مفتاح key2lix_client_orders_filter، استعادة عند التحميل |
| **عرض جوال محسّن** | تحسين عرض جدول الطلبات والبطاقات على الشاشات الصغيرة (بطاقات بدل جدول إن لزم). | متوسطة | ✅ منفّذ: @media (max-width: 768px) يعرض بطاقات الطلبات بدل الجدول |
| **دليل "كيف أتابع طلبي؟"** | رابط أو قسم قصير يشرح خطوات متابعة الطلب والمحادثة من حسابي. | قصيرة | ✅ منفّذ: قسم في تبويب طلباتي بثلاث خطوات + ترجمة ع/en |

## 6. التسجيل والدخول

| الاقتراح | الوصف | أولوية | الحالة |
|----------|--------|--------|--------|
| **تسجيل الدخول بـ Google/Apple** | تسجيل دخول اجتماعي (OAuth) اختياري. | طويلة | |
| **نسيت كلمة المرور** | استعادة كلمة المرور عبر البريد (رابط أو رمز مؤقت). | متوسطة | ✅ منفّذ: صفحة نسيت كلمة المرور، إرسال رابط إعادة التعيين، صفحة تعيين كلمة مرور جديدة |
| **تذكير "لم تؤكد بريدك"** | تنبيه واضح في حسابي وحتى منع الطلب حتى إكمال التأكيد (منفذ حالياً). | — | ✅ منفّذ: تنبيه في حسابي + منع الطلب حتى التأكيد |
| **إعادة إرسال رمز التأكيد** | زر "لم يصل الرقم؟ أعد الإرسال" مع حد زمني (مثلاً كل 60 ثانية). | قصيرة | ✅ منفّذ: زر في نموذج التأكيد، حد 60 ثانية، POST /api/client/resend-verify-email |

## 7. المفضلة والسلة

| الاقتراح | الوصف | أولوية | الحالة |
|----------|--------|--------|--------|
| **رابط المفضلة من حسابي** | روابط سريعة للمفضلة ومنتجاتي (منفذ جزئياً). | — | ✅ منفّذ: تبويب «المفضلة» في حسابي + رابط «فتح صفحة المفضلة»، وروابط سريعة في تبويب طلباتي |
| **"أضف كل المفضلة إلى السلة"** | زر في صفحة المفضلة يضيف كل المنتجات الحالية إلى السلة دفعة واحدة. | متوسطة | ✅ منفّذ: زر «أضف كل المفضلة إلى السلة» في صفحة /wishlist، يضيف المنتجات غير الموجودة في السلة مع تنبيه بعدد المضافة |
| **قوائم أمنيات متعددة** | إنشاء أكثر من قائمة (مثلاً "ألعاب"، "هدايا") مع إعادة تسمية وحذف. | منخفضة | — |

| الاقتراح | الوصف | أولوية | الحالة |
|----------|--------|--------|--------|
