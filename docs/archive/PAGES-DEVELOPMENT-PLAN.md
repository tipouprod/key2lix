## 2. نقاط القوة الحالية

- **توحيد معظم الصفحات:** استخدام `#navbar` و `#footer` مع partials، و`style.css` واحد، وثيم فاتح/داكن ولغة عربي/إنجليزي.
- **تجربة شراء كاملة:** منتجات → منتج → سلة → نموذج طلب → حسابي → محادثة.
- **لوحات محمية:** أدمن وبائع مع تبويبات ومنطق عمولة ومستحقات.
- **أداء وSEO:** تحميل مؤجل لـ search.js، صور lazy، WebP عند الرفع، CSP، إلخ.
- **إمكانية وصول:** ARIA، تخطي رابط، معالم، زر العودة للأعلى.

## 3.1 ما تم تنفيذه (حسب الفجوات أعلاه)

|--------|----------|

| **الإعلانات و API** | إعادة بناء ads.html و api.html بهيكل الموقع (navbar، footer، hero)؛ محتوى موجز مع روابط "سجّل كبائع" و/أو "دخول البائع" واتصل بنا؛ إضافة مفاتيح ترجمة adsTitle، adsDesc، apiTitle، apiDesc. |

## 4. خطط تطوير مقترحة (مراحل)

### المرحلة 1 — توحيد وإصلاحات سريعة (قصيرة الأمد)

| # | المهمة | الوصف |
|---|--------|--------|
| 1.1 | **توحيد صفحة Contact** | إزالة الـ navbar والستايلات المكررة من contact.html؛ الاعتماد على `#navbar` و `#footer` و style.css؛ الإبقاء على نموذج Contact وربط EmailJS فقط داخل الصفحة. |
| 1.2 | **إدراج Key2lix Plus في هيكل الموقع** | إضافة `#navbar` و `#footer` و style.css و lang.js و common.js لـ Key2lix-plus.html؛ إما محتوى "قريباً" أو نص توضيحي موحّد حتى يُضاف المحتوى الكامل. |
| 1.3 | **توحيد صفحات "تحت الإنشاء"** | قالب واحد (مثل skins/hardware/software): عنوان واضح، رسالة قصيرة، أزرار رئيسية + واتساب + ربما روابط فئات. استخدام نفس الـ hero أو صنف مشترك (مثل `.hero-under-construction`) لجميعها. |
| 1.4 | **مراجعة روابط الفوتر** | التأكد من صحة روابط الخصوصية والشروط؛ استبدال "#" لوسائل التواصل بروابط حقيقية أو إخفاء حتى تُضاف. |
