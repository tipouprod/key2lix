# اقتراحات مستقبلية ومطوّرة — لوحة البائع (Vendor Dashboard)

وثيقة مقترحات لتطوير لوحة البائع `/vendor` بناءً على الوضع الحالي: نظرة عامة، منتجاتي، طلباتي، تقارير، مستحقاتي.

---

## الوضع الحالي (ملخص)

| الميزة | الحالة |
|--------|--------|
| نظرة عامة | إحصائيات (منتجات، طلبات)، إجراءات سريعة، أفضل منتجات مبيعاً، نصيحة قابلة للإخفاء |
| منتجاتي | إضافة/تعديل/حذف، فئات، أسعار متعددة، صورة واحدة، جدول بحالة (معتمد/قيد المراجعة/مرفوض) |
| طلباتي | جدول مع فلتر (الكل/قيد التنفيذ/مكتمل)، تاريخ توصيل متوقع، محادثة، تعليم مكتمل |
| التقارير | أفضل منتجات مبيعاً، مبيعات حسب اليوم (رسم بسيط) |
| مستحقاتي | ملخص (مستحق/مدفوع/متبقي)، قائمة دفعات |

## 1. تجربة المستخدم والواجهة (UX/UI)

| الاقتراح | الوصف | أولوية |
|----------|--------|--------|
| **حفظ التبويب الأخير** | حفظ آخر تبويب مفتوح في `localStorage` وإعادة فتحه عند العودة. | قصيرة |
| **اختصارات لوحة المفاتيح** | مثلاً `1–5` للتبويبات، `N` لإضافة منتج جديد. | متوسطة |
| **بحث في جدول المنتجات** | حقل بحث (بالاسم أو الفئة) مع تصفية فورية. | قصيرة |
| **ترقيم الصفحات أو التحميل التدريجي** | عند كثرة المنتجات/الطلبات: pagination أو "تحميل المزيد". | متوسطة |
| **وضع مضغوط (compact) للجداول** | خيار لعرض صفوف أكثر في الشاشة. | منخفضة |
| **لوحة تحكم قابلة للتخصيص** | سحب وإفلات البطاقات في النظرة العامة أو إخفاء/إظهار أقسام. | طويلة |
| **ثيم مخصص للوحة البائع** | دعم ثيم داكن/فاتح مستقل عن الموقع إن رغب البائع. | منخفضة |

## 2. إدارة المنتجات

| الاقتراح | الوصف | أولوية |
|----------|--------|--------|
| **رفع عدة صور** | دعم أكثر من صورة لكل منتج (مع تحديد الصورة الرئيسية). | ✅ منفذ — حقل `images` متعدد، API يقبل حتى 10 صور. |
| **معاينة قبل الإرسال** | معاينة بطاقة المنتج قبل "إضافة المنتج". | ✅ منفذ — زر "معاينة" ونافذة تعرض الاسم، الفئة، السعر، والصورة المختارة. |
| **نسخ منتج** | زر "نسخ" لإنشاء منتج جديد انطلاقاً من منتج موجود (تعديل الـ slug ثم الحفظ). | ✅ منفذ — زر نسخ في جدول المنتجات يملأ النموذج بـ slug-copy ويُمرّر للصور. |
| **استيراد من CSV/Excel** | رفع ملف لاستيراد عدة منتجات دفعة واحدة (يتطلب API وتحققات). | طويلة |
| **وسوم وخصائص إضافية** | وسوم (tags) أو خصائص (مثلاً منصة اللعبة، المنطقة) للفلترة والبحث. | متوسطة | ✅ منفّذ: حقل وسوم (مفصولة بفاصلة) في إضافة/تعديل المنتج، والموقع يفلتر بالوسم |
| **تخفيضات ووقت محدود** | حقل "سعر قديم" و"نسبة خصم" أو "عرض حتى تاريخ" وعرضها في الموقع. | متوسطة | ✅ منفّذ: خصم، سعر قديم، عرض حتى تاريخ في النموذج؛ عرض في صفحة المنتج والقوائم |
| **حالة "مؤرشف"** | أرشفة منتج (إخفاء من القائمة دون حذف) لإعادة تفعيله لاحقاً. | متوسطة | ✅ منفّذ: حالة "مؤرشف" مستبعدة من القائمة العامة؛ أزرار أرشفة/استعادة في لوحة البائع |
| **تحذير تكرار الـ slug** | عند الإضافة/التعديل التحقق من عدم تكرار (فئة/فئة فرعية/slug). | قصيرة | ✅ منفّذ: GET check-slug + تحذير تحت حقل المفتاح عند التكرار |

## 3. الطلبات والتواصل

| الاقتراح | الوصف | أولوية |
|----------|--------|--------|
| **إشعارات الطلبات الجديدة** | إشعار في لوحة البائع أو بريد/واتساب عند طلب جديد على أحد منتجات البائع. | متوسطة |
| **عداد "طلبات بلا رد"** | في النظرة العامة أو شريط علوي: عدد الطلبات التي لم يرد عليها البائع (مثلاً خلال 24 ساعة). | قصيرة |
| **فلتر الطلبات بالتاريخ** | فلتر من–إلى تاريخ بالإضافة لفلتر الحالة. | قصيرة |
| **تصدير الطلبات (CSV/Excel)** | تصدير قائمة الطلبات للفترة المحددة. | متوسطة |
| **رد سريع من لوحة البائع** | إرسال رسالة من لوحة البائع دون الانتقال لصفحة المحادثة (نافذة منبثقة أو قسم مضمّن). | ✅ منفذ — زر "رد سريع" في كل طلب يفتح نافذة إرسال رسالة عبر POST /api/order/:id/messages. |
| **قوالب ردود** | قوالب جاهزة للرد على العميل (شكر، تأكيد استلام، تعليمات الدفع…). | ✅ منفذ — قائمة منسدلة في نافذة الرد السريع (شكراً لطلبك، تم استلام الطلب، تعليمات الدفع، تم الإرسال). |
| **تعليم "قيد التجهيز"** | حالة وسيطة بين قيد التنفيذ ومكتمل (مثلاً "قيد التجهيز" قبل "تم الإرسال"). | ✅ منفذ — حالة `preparing`، PATCH /api/vendor/orders/:orderId/status، زر "قيد التجهيز" للطلبات المعلقة، وعرض الحالة في حساب العميل. |

## 4. التقارير والتحليلات

## 10. الأمان والامتثال

| الاقتراح | الوصف | أولوية |
|----------|--------|--------|
| **تسجيل خروج من جميع الأجهزة** | خيار "تسجيل خروج من كل الجلسات" وإبطال tokens/جلسات أخرى. | متوسطة | ✅ منفّذ: عمود `logout_all_before`، تحقق في `requireVendor`، زر في إعداداتي + POST `/api/vendor/logout-all` |
| **سجل نشاط البائع** | سجل قراءة فقط (آخر تسجيل دخول، آخر تعديل منتج، إلخ) لأغراض الشفافية. | طويلة | ✅ منفّذ: جدول `vendor_activity_log`، تسجيل أحداث (دخول، خروج، أرشفة، منتج، ملف، 2FA)، عرض في تبويب إعداداتي |
| **تحقق ثنائي (2FA)** | اختياري للبائع لتعزيز أمان الحساب. | طويلة | ✅ منفّذ: TOTP (speakeasy)، إعداد QR من إعداداتي، تحقق عند الدخول، تعطيل بكلمة المرور |

## أولوية تنفيذ مقترحة

1. **قصيرة المدى (تحسينات سريعة)**  
   حفظ التبويب الأخير، بحث في المنتجات، معاينة المنتج قبل الإرسال، عداد طلبات بلا رد، فلتر الطلبات بالتاريخ، اختيار فترة في التقارير، تحذير تكرار slug، نسخ منتج.

2. **متوسطة المدى**  
   رفع عدة صور، وسوم/خصائص، تخفيضات، إشعارات الطلبات، قوالب ردود، رسم تقارير أوضح، تفصيل المستحقات حسب طلب، إعدادات البائع (الملف/كلمة المرور)، جرس إشعارات في اللوحة، تصدير طلبات/تقرير، PWA أو واجهة موبايل محسّنة، تسجيل خروج من جميع الأجهزة.

3. **طويلة المدى**  
   استيراد CSV/Excel، رد سريع من اللوحة، API للمورد واستيراد عبر API، Webhooks، إشعارات push، تحقق ثنائي، سجل نشاط، لوحة قابلة للتخصيص، مقارنة فترات وتقارير متقدمة.
