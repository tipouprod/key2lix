# مراجعة مستقبلية — الشكل والبنية التحتية | Key2lix

**وثيقة استراتيجية وتقنية** لقياس الوضع الحالي، توحيد الأولويات، وتمهيد الطريق للنمو والتميّز.  
*الإصدار: شباط 2026*

---

## خلاصة تنفيذية

| البُعد | الوضع الحالي | الاتجاه الاستراتيجي |
|--------|----------------|----------------------|
| **الشكل (UI/UX)** | واجهة احترافية، RTL، ثيم، PWA، وإمكانية وصول جيدة | تعميق **التحويل** (Wishlist، مراجعات، إشعارات)، **الأداء المُدرك** (skeleton، صور متجاوبة)، و**الهوية البصرية** القابلة للتخصيص. |
| **البنية التحتية** | Monolith Express منظم، SQLite/PostgreSQL، جلسات، أمان، وتكاملات سحابية | **قابلية الصيانة** (تقسيم routes/services)، **المراقبة والأمان** (audit، monitoring، rate limit ديناميكي)، و**الاستعداد للتوسع** (S3، reverse proxy، عقد متعددة). |
| **API والتكامل** | جلسات، نسخ API موثّقة، إصدار v1 مفعّل | **إصدار API واضح** (v1/v2)، ومستقبلاً **مفاتيح API للبائعين** و**Webhooks** لربط الأنظمة. |

الوثيقة تُحدّث دورياً وتُستخدم مع `IMPLEMENTATION-ROADMAP.md` و `PROFESSIONAL-SUGGESTIONS.md` و `MOBILE-UX.md` لتوحيد الخطط واتخاذ القرار.

## 1. الوضع الحالي — تقييم موجز

### 1.1 الشكل (UI/UX)
- **التقنية:** Vanilla HTML/CSS/JS بدون إطار frontend — خفيف وسريع، مع إمكانية ترحيل لاحق إلى مكونات إذا لزم.
- **التصميم:** متغيرات CSS (ألوان بنفسجي، سطح شفاف، ظلال)، خط Outfit، RTL وعربي كامل، ثيم فاتح/داكن.
- **الصفحات:** رئيسية (هيرو، إحصائيات، فئات، عروض، فلاش)، منتجات، منتج، سلة، تسجيل/دخول، حساب عميل، لوحة بائع، لوحة أدمن، دعم، تواصل، صفقات، وأقسام (hardware/software/subscriptions).
- **الموبايل:** Safe area، Bottom Nav (5 عناصر)، درج جانبي، PWA (manifest + Service Worker).
- **إمكانية الوصول:** skip-link، focus-visible، تسميات ARIA، دعم لوحة المفاتيح.

### 1.2 البنية التحتية
- **الخادم:** Express 5، نقطة دخول واحدة `server.js`؛ routes الصفحات في `routes/pages.js`.
- **قاعدة البيانات:** SQLite (better-sqlite3) افتراضيًا؛ دعم PostgreSQL عبر `DB_DRIVER=postgres` و `database/db-pg.js`.
- **المصادقة:** جلسات (express-session)، أدمن/بائع/عميل منفصلون، 2FA للبائع (speakeasy + QR).
- **الأمان:** Helmet (CSP، HSTS في الإنتاج)، rate limits، Brotli + gzip.
- **التكاملات:** Stripe، Nodemailer/EmailJS، Web Push، OpenAI (اختياري)، Sentry، نسخ احتياطي محلي وسحابي.

## 3. أفكار مستقبلية — البنية التحتية

### 3.1 تقسيم الكود وقابلية الصيانة
| الفكرة | الوصف | الأولوية |
|--------|--------|----------|
| **تقسيم Routes** ✅ | مسارات الصفحات في `routes/pages.js` مع `registerPages(app, opts)`؛ API/أدمن/بائع يمكن نقلها لاحقاً إلى routes مخصّصة. | عالية |
| **طبقة خدمات (Services)** ✅ | منطق العمولة في `services/commissionService.js` (refreshConfig، computeCommission، computeFinalPrice، إلخ). | عالية |
| **Middleware منفصل** ✅ | مصادقة الأدمن والبائع في `middleware/auth.js` (requireAdmin، requireVendor، التحقق من الجلسة وحالة المورد). | متوسطة |
| **تحقق (Validation) مركزي** ✅ | قواعد express-validator في `validators/order.js` و `validators/contact.js` وإعادة استخدامها في server.js. | متوسطة |

### 3.2 قاعدة البيانات والأداء
| الفكرة | الوصف | الأولوية |
|--------|--------|----------|
| **الانتقال الكامل لـ PostgreSQL في الإنتاج** ✅ | `DB_DRIVER=postgres` و `DATABASE_URL`؛ ترحيل وجداول وفهارس موثّقة في [POSTGRES-MIGRATION.md](POSTGRES-MIGRATION.md). | عالية عند التوسع |
| **فهارس إضافية** ✅ | فهارس في SQLite وPostgreSQL: orders، products، reviews، notifications. | متوسطة |
| **قراءة/كتابة منفصلة (Read replicas)** | Replica للقراءة فقط للتقارير والقوائم؛ موثّق في POSTGRES-MIGRATION.md. | منخفضة (مستقبلي) |
| **تخزين الجلسات خارج الذاكرة** ✅ | **SESSION_STORE=db** — جدول `sessions` عبر `lib/session-store-db.js`؛ مناسب لمتعدد العقد. | متوسطة عند التوسع |

### 3.3 التخزين والملفات
| الفكرة | الوصف | الأولوية |
|--------|--------|----------|
| **رفع الصور إلى S3 (أو بديل)** ✅ | **IMAGES_S3_BUCKET** و **IMAGES_S3_PUBLIC_URL_BASE**: رفع صور المنتجات إلى S3/Backblaze B2 بعد المعالجة. [IMAGES-S3.md](IMAGES-S3.md). | عالية عند النمو |
| **معالجة صور تلقائية** ✅ | أحجام متعددة (thumbnail 300px، medium 800px، large 1920px) و WebP عبر Sharp؛ تخزين [thumb, medium, main] في images_json. | عالية |
| **نسخ احتياطي قاعدة البيانات** ✅ | نسخ محلي + رفع سحابي (BACKUP_UPLOAD_ENABLED)؛ جدولة أوقات منخفضة الحمل؛ [BACKUP.md](BACKUP.md). | منخفضة (معزّز) |

### 3.4 الأمان والمراقبة
| الفكرة | الوصف | الأولوية |
|--------|--------|----------|
| **تسجيل خروج تلقائي مع تمديد** ✅ | **SESSION_INACTIVITY_MINUTES**: انتهاء الجلسة بعد عدم النشاط؛ تحديث lastActivity عند كل طلب؛ حد أقصى من cookie.maxAge (24 ساعة). | متوسطة |
| **لوحة مراقبة بسيطة للأدمن** ✅ | تبويب "المراقبة": **GET /api/admin/monitoring** — حالة DB، Uptime، نسخ احتياطي، آخر 10 أخطاء. | عالية |
| **تدقيق (Audit) حرج** ✅ | تسجيل: موافقة/رفض مورد، تغيير حالة طلب، عمولة، حذف، رفض منتج، ثيم. **GET /api/admin/audit-log** وعرض في المراقبة. | متوسطة |
| **Rate limit ديناميكي** ✅ | **GET/POST /api/admin/settings/rate-limit** — تخزين الحدود في settings؛ تطبيق فوري دون إعادة تشغيل. | منخفضة |

### 3.5 التوسع والتوفر
| الفكرة | الوصف | الأولوية |
|--------|--------|----------|
| **تفعيل Brotli للثابت** ✅ | **lib/compression-brotli.js**: ضغط Brotli عند Accept-Encoding: br؛ تعاون مع compression (gzip). | متوسطة |
| **Reverse proxy و Cache** ✅ | Nginx أو Caddy أمام Node: تقديم الثابت مع cache وإنهاء SSL. **مُطبَّق:** ملفات جاهزة في `config/nginx-key2lix.example.conf` و `config/Caddyfile.example`. [REVERSE-PROXY.md](REVERSE-PROXY.md). | عالية عند الإنتاج الكبير |
| **بيئة متعددة العقد** ✅ | تشغيل أكثر من عملية Node: **pm2 start ecosystem.config.cjs** أو **npm run start:cluster**؛ **SESSION_STORE=db** للجلسات المشتركة. [SCALING-FUTURE.md](SCALING-FUTURE.md). | منخفضة (مستقبلي) |
| **Queue للعمليات الثقيلة** ✅ | **البريد:** Bull + Redis — ضبط **REDIS_URL** و **QUEUE_ENABLED=1** ينقل إرسال البريد إلى طابور. معالجة الصور/التقارير لاحقاً. [QUEUE.md](QUEUE.md)، [SCALING-FUTURE.md](SCALING-FUTURE.md). | منخفضة (معزّز) |

## 5. توجهات مبتكرة وممارسات حديثة

- **التحويل أولاً:** ترتيب أولويات الشكل حول تحسين التحويل (سلة، أمنيات، مراجعات، إشعارات) وليس الشكل فقط.
- **الأداء المُدرك:** Skeleton و responsive images و Brotli لتحسين الإحساس بالسرعة حتى قبل تحسين الخادم.
- **المراقبة والأمان كأساس:** مراقبة بسيطة، audit log، و rate limit ديناميكي لتمكين النمو دون مفاجآت.
- **إصدار API واضح:** دعم /api/v1 من اليوم يمهّد لـ v2 دون كسر العملاء.
- **التخزين والجلسات للتوسع:** S3 للصور وجلسات في DB تمكّنان عقد متعددة ونسخ احتياطي أفضل.
- **توثيق موحّد:** وثائق منفصلة (POSTGRES، BACKUP، IMAGES-S3، REVERSE-PROXY، SCALING-FUTURE، API-INTEGRATION-FUTURE) لاتخاذ قرارات سريعة وتدريب الفريق.

## 6. خلاصة واستخدام الوثيقة

- **الشكل:** واجهة احترافية ومتجاوبة مع RTL وثيم وموبايل. الأولوية للتحويل (Wishlist، مراجعات، إشعارات، Sticky CTA)، الأداء المُدرك (skeleton، responsive images)، والهوية (فلاتر، Trust، محتوى مورد).
- **البنية التحتية:** Monolith منظم؛ الأولوية لتقسيم الكود (routes، services، middleware)، ثم التخزين والصور، والمراقبة، والاستعداد لـ PostgreSQL و reverse proxy عند النمو.

**استخدام الوثيقة:** مع `IMPLEMENTATION-ROADMAP.md` و `PROFESSIONAL-SUGGESTIONS.md` و `MOBILE-UX.md` لتوحيد الخطط، مراجعة ربع سنوية، وتحديث الجدول والأولويات حسب الإنجاز والسوق.
