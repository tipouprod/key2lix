# Queue البريد (Bull + Redis) — Key2lix

عند تفعيل Redis و Queue، تُضاف مهام البريد (إشعارات، تحقق، استعادة كلمة المرور، إلخ) إلى طابور Bull بدل إرسالها داخل طلب الـ API، لعدم حجب الاستجابة.

---

## التفعيل

في `.env`:

- **REDIS_URL** — رابط Redis (مثل `redis://127.0.0.1:6379`) أو
- **REDIS_HOST** واختياري **REDIS_PORT** (افتراضي 6379)
- **QUEUE_ENABLED=1**

مثال:

```env
REDIS_URL=redis://127.0.0.1:6379
QUEUE_ENABLED=1
```

---

## التبعيات

يحتاج المشروع إلى `bull` و `ioredis` (مثبتان في package.json). شغّل Redis محلياً أو استخدم خدمة سحابية.

---

## السلوك

- عند استدعاء إرسال بريد (طلب جديد، اعتماد منتج، تحقق بريد، استعادة كلمة مرور، تغيير حالة طلب، رد على محادثة، نشرية، تنبيه منتج): إن كان Queue مفعّلاً تُضاف المهمة إلى طابور `key2lix-email` ويُرجع الـ API فوراً.
- Worker يعمل في نفس عملية السيرفر (يُشغّل عند الـ listen) ويستخرج المهام ويرسل البريد عبر `lib/email`.
- إعادة محاولة: 3 محاولات مع تأخير تصاعدي.

---

## عقد متعددة

عند تشغيل أكثر من عملية (مثلاً PM2 أو `npm run start:cluster`)، كل عملية تشغّل worker خاصاً بها؛ Bull يوزّع المهام على العمال. ضع **SESSION_STORE=db** للجلسات عند استخدام عقد متعددة.
