# استكشاف أخطاء ngrok

عندما **ngrok لا يعمل**، راجع النقاط التالية بالترتيب.

## 1. التوكن (NGROK_AUTHTOKEN)

- يجب أن يكون في ملف **`.env`** (وليس فقط في `.env.example`):
  ```env
  NGROK_AUTHTOKEN=الرقم_الطويل_الذي_حصلت_عليه
  ```
- احصل على التوكن من:  
  **https://dashboard.ngrok.com/get-started/your-authtoken**
- انسخ التوكن كما هو، بدون مسافات قبل أو بعد.
- إذا كان لديك **NGROK_DOMAIN** (نطاق ثابت مدفوع)، تأكد أنه مسجّل في حسابك في لوحة ngrok.

## 2. المنفذ (PORT)

- السيرفر يجب أن يعمل على نفس المنفذ الذي ينتظر عليه ngrok.
- الافتراضي في المشروع: **3000**. إذا غيّرت `PORT` في `.env`، تأكد أن السيرفر يشغّل على هذا المنفذ.
- عند استخدام **تشغيل السيرفر + النفق معاً** (`npm run start:tunnel`): السكربت يشغّل السيرفر ثم يفتح النفق على نفس المنفذ تلقائياً.

## 3. تشغيل السيرفر قبل النفق (عند استخدام `npm run tunnel` فقط)

- لو استخدمت **`npm run tunnel`** فقط (بدون `start:tunnel`):
  1. في طرفية أولى: `npm start` (أو `node server.js`).
  2. بعد أن يظهر أن السيرفر يعمل، في طرفية ثانية: `npm run tunnel`.

## 4. جدار الحماية / الشبكة

- بعض شبكات العمل أو الجامعة أو مزود الإنترنت تحجب اتصال ngrok.
- جرّب من شبكة أخرى (مثلاً هاتفك كمودم) أو تعطيل VPN مؤقتاً.
- تأكد أن جدار حماية Windows (أو أي برنامج حماية) لا يمنع Node أو ngrok من الاتصال بالإنترنت.

## 5. إصدار الحزمة وإعادة التثبيت

- المشروع يستخدم **@ngrok/ngrok** (SDK لـ Node).
- تأكد من تثبيت الحزم:
  ```bash
  npm install
  ```
- إذا استمر الخطأ، جرّب تحديث الحزمة:
  ```bash
  npm update @ngrok/ngrok
  ```

## 6. رسالة الخطأ الشائعة

| الرسالة | ما تفعله |
|--------|----------|
| `NGROK_AUTHTOKEN غير معرّف` | أضف التوكن في `.env` كما في النقطة 1. |
| `failed to start tunnel` أو `tunnel session failed` | تحقق من صحة التوكن، ومن الشبكة وجدار الحماية (4، 5). |
| `domain ... is not available` | النطاق الثابت (NGROK_DOMAIN) إما غير مسجّل في حسابك أو مستخدم — راجع لوحة ngrok. |
| `address already in use` | المنفذ مستخدم. أوقف العملية التي تستخدم PORT أو غيّر PORT في `.env`. |

## 7. سكربت التشخيص

- من جذر المشروع شغّل:
  ```bash
  npm run check:ngrok
  ```
  أو:
  ```bash
  node scripts/check-ngrok.js
  ```
  يعرض هل التوكن معرّف وصحيح، ثم يحاول فتح النفق ويعرض **رسالة الخطأ الكاملة** إن فشل (مهم لمعرفة السبب).

## 8. اختبار سريع من الطرفية

- التأكد من أن التوكن يقرأ من `.env`:
  ```bash
  node -e "require('dotenv').config(); console.log('TOKEN exists:', !!process.env.NGROK_AUTHTOKEN);"
  ```
  يجب أن يطبع: `TOKEN exists: true`.

بعد تطبيق هذه الخطوات، إن استمرت المشكلة شغّل `npm run check:ngrok` وانسخ **رسالة الخطأ الكاملة** من الطرفية (أو راجع وثائق ngrok: https://ngrok.com/docs).

## 9. صفحة بيضاء بعد "Visit Site"

عند فتح الرابط عبر ngrok والضغط على "Visit Site" ثم لا يظهر شيء:

- **ما تم تطبيقه في المشروع:** عند الطلب من نطاق ngrok، السيرفر يقدّم الصفحة الرئيسية (`/`) مع محمّل يفتح سكربتات الصفحة عبر `fetch` مع الهيدر `ngrok-skip-browser-warning` حتى لا تعيد ngrok صفحة التحذير بدل الملفات. كما أن طلبات `fetch` من الواجهة تُزوَّد تلقائياً بهذا الهيدر عند العمل عبر ngrok.
- **إن استمرت صفحة بيضاء أو "لا يجلب شيء":**
  1. افتح أدوات المطوّر (F12) → تبويب **Console** وابحث عن أخطاء مثل `ngrok CSS fail` أو `ngrok JS fail` لمعرفة إن كانت الطلبات تفشل.
  2. جرّب تحديث الصفحة (F5) مرة أخرى بعد "Visit Site"، أو افتح الرابط في نافذة خاصة (Incognito).
  3. **إضافة المتصفح:** ثبّت إضافة مثل **ModHeader** (Chrome/Edge) وأضف هيدر الطلب: الاسم `ngrok-skip-browser-warning` والقيمة `1` للنطاق `*.ngrok-free.dev` أو نطاقك، ثم أعد تحميل الصفحة.
  4. **طلبات fetch/API من كودك إلى رابط ngrok:** يجب أن يتضمّن الطلب هيدر `ngrok-skip-browser-warning: 1` وإلا سترجع ngrok صفحة التحذير بدل المحتوى. مثال:
     ```js
     fetch("https://YOUR-SUBDOMAIN.ngrok-free.dev/", {
       method: "GET",
       headers: { "ngrok-skip-browser-warning": "1" },
       credentials: "omit"  // أو "include" حسب الحاجة
     });
     ```
  5. إن لم ينجح، قد تحتاج باقة ngrok مدفوعة لإزالة صفحة التحذير.

- **تحذيرات Console عن خطوط (assets.ngrok.com/fonts/...):** هذه من **صفحة ngrok** ("Visit Site") وليست من موقعك. يمكن تجاهلها؛ بعد الضغط على "Visit Site" والانتقال لـ Key2lix تختفي لأنك لم تعد على صفحة ngrok.

## 10. ERR_NGROK_804 — إعادة تشغيل عن بُعد غير مدعومة

عند ظهور:  
`ERR_NGROK_804 [invalid_argument] This ngrok agent does not support remote restarting`  
معناه أن لوحة ngrok (أو واجهة أخرى) حاولت إعادة تشغيل الوكيل عن بُعد، والوكيل الحالي لم يُشغَّل بطريقة تدعم ذلك (مثلاً عند استخدام السكربت `npm run start:tunnel` أو `npm run tunnel`).

**ما تفعله:** لا تستخدم زر "Restart" أو إعادة التشغيل من لوحة ngrok. لإعادة تشغيل النفق أوقف العملية محلياً (**Ctrl+C** في الطرفية) ثم شغّلها من جديد:
- `npm run start:tunnel` أو
- `npm start` في طرفية و `npm run tunnel` في أخرى.
