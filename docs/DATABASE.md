# قاعدة البيانات — Key2lix

## الملف والمسار

- **المسار الافتراضي:** `client/data/key2lix.db` (نسبةً لجذر المشروع)
- **بعد التبديل من keylix إلى key2lix:** إذا وُجد ملف `keylix.db` في `client/data`، التطبيق **يستخدمه تلقائياً** عند عدم تعيين `DB_FILENAME`، حتى لا تختفي البيانات أو منتجات الموردين. يمكنك التأكد من اللوج عند التشغيل: `Database ready` يعرض المسار الفعلي (قد يكون `.../keylix.db`).
- **استخدام ملف معيّن:** لفرض ملف معين أضف في `.env`: `DB_FILENAME=keylix.db` أو `DB_FILENAME=key2lix.db`.
- **الترحيل التلقائي:** عند التشغيل، التطبيق ينقل تلقائياً من ملفات JSON (إن وُجدت) إلى قاعدة البيانات:
  - المنتجات: `products.json` أو `products.json.migrated`
  - الطلبات: `orders.json` أو `orders.json.migrated`
  - جهات الاتصال: `contacts.json` أو `contacts.json.migrated`
  - العملاء: `clients.json` أو `clients.json.migrated`
  - الموردون: `vendors.json` أو `vendors.json.migrated`
  - العملاء من الطلبات: إذا كان جدول العملاء فارغاً والطلبات تحتوي بريد إلكتروني، يُنشأ عميل لكل بريد فريد ويُرتبط بالطلبات. بذلك تُخزَّن كل البيانات في قاعدة البيانات وتُعرض منها.
- **النوع:** SQLite (better-sqlite3)

## لا يظهر عملاء أو موردون

- **العملاء:** يُنشَؤون تلقائياً من الطلبات (كل بريد في الطلبات → عميل). عند كل تشغيل للسيرفر، إن كان جدول العملاء فارغاً والطلبات تحتوي بريداً، يُعاد تشغيل الترحيل. تأكد أن الحزمة `bcrypt` مثبتة: `npm install bcrypt`.
- **الموردون:** يُضافون عبر التسجيل من الموقع (`/vendor-register`) أو من لوحة الأدمن، أو من ملف `vendors.json` / `vendors.json.migrated` إن وُجد في `client/data`. إن لم يكن لديك ملف موردين، أضف مورداً من لوحة الأدمن أو من صفحة تسجيل المورد.

## إنشاء قاعدة البيانات تلقائياً

**قاعدة البيانات لا تُنشأ يدوياً.** عند تشغيل السيرفر لأول مرة (`npm start`):

1. يُنشأ المجلد `client/data` إن لم يكن موجوداً.
2. يُنشأ ملف `key2lix.db` ويُهيَّأ بجميع الجداول والإ migrations.

إذا ظهرت رسالة مثل **«قاعدة البيانات غير موجودة»** (مثلاً عند إنشاء نسخة احتياطية):

- تأكد أنك شغّلت السيرفر مرة واحدة على الأقل: `npm start`
- تأكد أن المجلد `client/data` قابل للكتابة من قبل عملية Node
- بعد التشغيل ستجد الملف: `client/data/key2lix.db`
