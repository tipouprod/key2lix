# خطوة بناء الواجهة — Key2lix

## ماذا يفعل البناء؟

- ينسخ مجلد `client/` بالكامل إلى `dist/`.
- يصدف (يُصغّر) كل ملفات **JavaScript** في `client/assets/js/` باستخدام **terser**.
- يصدف ملف **CSS** الرئيسي `client/assets/css/style.css` باستخدام **clean-css**.
- الناتج في مجلد `dist/` جاهز للنشر مع أحجام أصغر وأداء أفضل.

## الأوامر

```bash
# بناء الواجهة (ينتج مجلد dist/)
npm run build
```

## تشغيل السيرفر من النسخة المُصدّفة

بعد تشغيل `npm run build`، يمكن جعل السيرفر يخدم الملفات من `dist/` بدل `client/`:

```bash
# تشغيل عادي (يخدم من client/)
npm start

# تشغيل مع النسخة المُصدّفة (يخدم من dist/)
USE_BUILD=1 npm start
```

- **Windows (PowerShell):** `$env:USE_BUILD="1"; node server.js`
- تأكد من وجود مجلد `dist/` (أي أنك نفذت `npm run build` مرة واحدة على الأقل قبل استخدام `USE_BUILD=1`).

## ملاحظات

- مجلد `dist/` مضاف في `.gitignore` ولا يُرفع إلى المستودع.
- في بيئة الإنتاج يُفضّل تشغيل `npm run build` ثم `USE_BUILD=1 npm start` (أو ضبط المتغير في بيئة التشغيل).

## خطط مستقبلية (للعمل عليها)

- دعم مصدر خرائط المصدر (source maps) عند التصغير لتسهيل تتبع الأخطاء في الإنتاج.
- اختياري: دمج خطوة بناء مع bundler (مثل esbuild) لتقسيم الحزم وتقليل الحجم الأولي.
