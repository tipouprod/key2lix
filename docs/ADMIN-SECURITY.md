# تأمين لوحة الأدمن — Key2lix

ملخص ما هو مُنفَّذ لتعزيز أمان لوحة الأدمن ومساراتها.

## ما تم تنفيذه

| الميزة | الوصف |
|--------|--------|
| **حماية CSRF** | توكن CSRF يُولَّد مع الجلسة ويُتحقَّق منه في كل طلب POST/PUT/PATCH/DELETE إلى `/api/admin/*` (ما عدا تسجيل الدخول و2FA). الواجهة تضيف رأس `X-CSRF-Token` تلقائياً. |
| **رؤوس أمان** | لمسارات `/admin`, `/login`, `/api/admin`, `/api/login`: `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Referrer-Policy: strict-origin-when-cross-origin`. |
| **ربط الجلسة** | عند الدخول تُحفظ في الجلسة: عنوان IP و hash لـ User-Agent. في كل طلب أدمن يُتحقَّق من تطابقهما؛ عند الاختلاف تُنهى الجلسة. |
| **انتهاء صلاحية الجلسة** | حد أقصى لعمر الجلسة (ساعات)، افتراضي 8. يُضبط عبر `ADMIN_SESSION_MAX_AGE_HOURS`. |
| **تسجيل خروج من كل الأجهزة** | إعداد `admin_sessions_invalid_before` في قاعدة البيانات يبطل كل الجلسات الأقدم من ذلك الوقت. زر في الواجهة (للأدمن الرئيسي فقط) ونداء `POST /api/admin/logout-all`. |
| **قائمة IP مسموحة** | اختياري: إن وُجد `ADMIN_IP_ALLOWLIST` يُرفض الوصول لمسارات الأدمن من أي IP غير مدرج (يدعم CIDR للـ IPv4). |
| **سجل التدقيق** | تسجيل إجراءات مثل: دخول من جهاز جديد، تغيير إعدادات، موافقة/رفض بائع، حذف طلب/عميل، نسخ احتياطي واستعادة، تسجيل خروج من كل الأجهزة، إلخ. |
| **تنبيه دخول من جهاز جديد** | عند أول دخول ناجح من عنوان IP لم يظهر في آخر 20 تسجيل دخول ناجح، يُسجَّل في سجل التدقيق إجراء `admin_login_new_device`. |

## متغيرات البيئة (.env)

| المتغير | الوصف | افتراضي |
|---------|--------|---------|
| `ADMIN_IP_ALLOWLIST` | قائمة عناوين IP أو نطاقات (مفصولة بفاصلة)، مثال: `1.2.3.4,10.0.0.0/8`. فارغ = السماح للجميع. | — |
| `ADMIN_SESSION_BIND_IP` | `1` = ربط الجلسة بعنوان IP، `0` = تعطيل. | مفعّل |
| `ADMIN_SESSION_BIND_UA` | `1` = ربط الجلسة بـ User-Agent، `0` = تعطيل. | مفعّل |
| `ADMIN_SESSION_MAX_AGE_HOURS` | أقصى عمر لجلسة الأدمن بالساعات؛ بعدها يُطلب تسجيل الدخول من جديد. | 8 |

## الملفات ذات الصلة

- **`middleware/admin-security.js`**: رؤوس أمان، قائمة IP، ربط الجلسة، CSRF، انتهاء الصلاحية.
- **`server.js`**: تطبيق الـ middleware، تعيين ربط الجلسة عند الدخول، نقاط النهاية `GET /api/admin/csrf-token` و `POST /api/admin/logout-all`.
- **`client/pages/admin.html`**: جلب توكن CSRF، إرسال الرأس مع الطلبات، زر «خروج من كل الأجهزة»، روابط سجل التدقيق في تبويب أمان وحماية.
- **`.env.example`**: تعليقات للمتغيرات أعلاه.

## واجهة الأدمن

- في **الإعدادات → أمان وحماية**: روابط «عرض سجل التدقيق» و«تصدير CSV»، ونص يوضح ربط الجلسة و«تسجيل خروج من كل الأجهزة».
- في **رأس الصفحة**: زر «خروج من كل الأجهزة» (يظهر للأدمن الرئيسي فقط)، بتنسيق مميز (لون عنبري).
